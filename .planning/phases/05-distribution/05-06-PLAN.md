  <name>Task 1: Create version synchronization check workflow</name>
  <files>.github/workflows/version-check.yml</files>
  <action>
Create a CI workflow that verifies all package versions match the Cargo.toml workspace version.

Workflow structure:
```yaml
name: Version Check

on:
  push:
    branches: [main]
  pull_request:
    branches: [main]
    paths:
      - 'Cargo.toml'
      - 'py/pyproject.toml'
      - 'js/package.json'
      - 'bindings/csharp/MarketdataUniffi/MarketdataUniffi.csproj'
      - 'bindings/java/build.gradle.kts'

jobs:
  check:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4

      - name: Extract workspace version
        id: workspace
        run: |
          VERSION=$(grep -A1 '\[workspace.package\]' Cargo.toml | grep 'version' | cut -d'"' -f2)
          echo "version=$VERSION" >> $GITHUB_OUTPUT
          echo "Workspace version: $VERSION"

      - name: Check Python version
        run: |
          PY_VERSION=$(grep '^version' py/pyproject.toml | cut -d'"' -f2)
          echo "Python version: $PY_VERSION"
          if [ "$PY_VERSION" != "${{ steps.workspace.outputs.version }}" ]; then
            echo "::error::Python version $PY_VERSION != workspace ${{ steps.workspace.outputs.version }}"
            exit 1
          fi

      - name: Check Node.js version
        run: |
          JS_VERSION=$(node -p "require('./js/package.json').version")
          echo "Node.js version: $JS_VERSION"
          if [ "$JS_VERSION" != "${{ steps.workspace.outputs.version }}" ]; then
            echo "::error::Node.js version $JS_VERSION != workspace ${{ steps.workspace.outputs.version }}"
            exit 1
          fi

      - name: Check C# version
        run: |
          CS_VERSION=$(grep '<Version>' bindings/csharp/MarketdataUniffi/MarketdataUniffi.csproj | sed 's/.*<Version>\(.*\)<\/Version>.*/\1/')
          echo "C# version: $CS_VERSION"
          if [ "$CS_VERSION" != "${{ steps.workspace.outputs.version }}" ]; then
            echo "::error::C# version $CS_VERSION != workspace ${{ steps.workspace.outputs.version }}"
            exit 1
          fi

      - name: Check Java version
        run: |
          JAVA_VERSION=$(grep 'version = ' bindings/java/build.gradle.kts | head -1 | cut -d'"' -f2)
          echo "Java version: $JAVA_VERSION"
          if [ "$JAVA_VERSION" != "${{ steps.workspace.outputs.version }}" ]; then
            echo "::error::Java version $JAVA_VERSION != workspace ${{ steps.workspace.outputs.version }}"
            exit 1
          fi

      - name: All versions match
        run: echo "All package versions match workspace version ${{ steps.workspace.outputs.version }}"
```

This prevents releases with version drift.

<task type="auto">
  <name>Task 2: Create release coordinator workflow</name>
  <files>.github/workflows/release.yml</files>
  <action>
Create the main release workflow triggered by git tags.

Workflow structure:
```yaml
name: Release

on:
  push:
    tags:
      - 'v*.*.*'

env:
  CARGO_TERM_COLOR: always

jobs:
  # Extract version from tag
  extract-version:
    runs-on: ubuntu-latest
    outputs:
      version: ${{ steps.version.outputs.version }}
    steps:
      - id: version
        run: |
          VERSION=${GITHUB_REF#refs/tags/v}
          echo "version=$VERSION" >> $GITHUB_OUTPUT
          echo "Release version: $VERSION"

  # Wave 1: Build all platforms in parallel
  build-python:
    needs: extract-version
    uses: ./.github/workflows/build-python.yml
    with:
      version: ${{ needs.extract-version.outputs.version }}

  build-nodejs:
    needs: extract-version
    uses: ./.github/workflows/build-nodejs.yml
    with:
      version: ${{ needs.extract-version.outputs.version }}

  build-uniffi:
    needs: extract-version
    uses: ./.github/workflows/build-uniffi.yml
    with:
      version: ${{ needs.extract-version.outputs.version }}

  # Wave 2: Publish after builds complete
  publish-python:
    needs: build-python
    uses: ./.github/workflows/publish-python.yml

  publish-nodejs:
    needs: build-nodejs
    uses: ./.github/workflows/publish-nodejs.yml
    secrets:
      NPM_TOKEN: ${{ secrets.NPM_TOKEN }}

  publish-nuget:
    needs: build-uniffi
    uses: ./.github/workflows/publish-nuget.yml
    secrets:
      NUGET_API_KEY: ${{ secrets.NUGET_API_KEY }}

  publish-java:
    needs: build-uniffi
    uses: ./.github/workflows/publish-java.yml

  # Wave 3: Create GitHub Release after all publishes
  create-release:
    needs: [publish-python, publish-nodejs, publish-nuget, publish-java]
    runs-on: ubuntu-latest
    permissions:
      contents: write
    steps:
      - uses: actions/checkout@v4

      - name: Generate release notes
        id: notes
        run: |
          # Extract changelog section for this version
          VERSION=${{ needs.extract-version.outputs.version }}
          echo "## What's Changed" > release_notes.md
          echo "" >> release_notes.md
          echo "### Packages Published" >> release_notes.md
          echo "- Python: \`pip install fugle-marketdata==$VERSION\`" >> release_notes.md
          echo "- Node.js: \`npm install @fugle/marketdata@$VERSION\`" >> release_notes.md
          echo "- C#: \`dotnet add package MarketdataUniffi --version $VERSION\`" >> release_notes.md
          echo "- Java: Add to build.gradle:" >> release_notes.md
          echo "  \`\`\`kotlin" >> release_notes.md
          echo "  implementation(\"tw.com.fugle:marketdata-java:$VERSION\")" >> release_notes.md
          echo "  \`\`\`" >> release_notes.md

      - uses: softprops/action-gh-release@v1
        with:
          body_path: release_notes.md
          generate_release_notes: true
```

Key points:
- Triggered by v*.*.* tags (e.g., v0.2.0)
- Wave 1: All builds run in parallel
- Wave 2: Publish jobs depend on their respective build jobs
- Wave 3: GitHub Release created after all publishes succeed
- Uses reusable workflows with `uses: ./.github/workflows/...`
