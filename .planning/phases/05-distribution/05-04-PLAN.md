  <name>Task 1: Create PyPI publish workflow with trusted publishing</name>

  <action>
Create a reusable workflow that publishes Python wheels to PyPI using trusted publishing (OIDC).

Workflow structure:
```yaml
name: Publish Python to PyPI

on:
  workflow_call:

jobs:
  publish:
    runs-on: ubuntu-latest
    environment: release  # Required for trusted publishing
    permissions:
      id-token: write     # CRITICAL: Required for OIDC
      contents: read
    steps:
      - uses: actions/download-artifact@v4
        with:
          pattern: wheels-*
          merge-multiple: true
          path: dist/

      - uses: pypa/gh-action-pypi-publish@v1.12.2
        with:
          packages-dir: dist/
          # No password/token - OIDC handles authentication
```

Key points:
- environment: release - must match PyPI trusted publisher config
- id-token: write - enables OIDC token generation
- merge-multiple: true - combines all wheel artifacts into dist/
- No NPM_TOKEN or PYPI_TOKEN needed with trusted publishing

Add a skip-existing option for re-runs:
```yaml
with:
  skip-existing: true
```

<task type="auto">
  <name>Task 2: Create npm publish workflow</name>
  <files>.github/workflows/publish-nodejs.yml</files>
  <action>
Create a reusable workflow that publishes Node.js packages to npm.

Workflow structure:
```yaml
name: Publish Node.js to npm

on:
  workflow_call:
    secrets:
      NPM_TOKEN:
        required: true

jobs:
  publish:
    runs-on: ubuntu-latest
    permissions:
      id-token: write  # For npm provenance
      contents: read
    steps:
      - uses: actions/checkout@v4

      - uses: actions/setup-node@v4
        with:
          node-version: 20
          registry-url: 'https://registry.npmjs.org'

      - uses: actions/download-artifact@v4
        with:
          pattern: bindings-*
          path: js/

      - name: Move bindings to correct location
        working-directory: js
        run: |
          for dir in bindings-*/; do
            mv "$dir"/*.node . 2>/dev/null || true
          done
          rm -rf bindings-*/
          ls -la *.node

      - name: Prepare package
        working-directory: js
        run: |
          npm ci
          npm run prepublishOnly

      - name: Publish with provenance
        working-directory: js
        run: npm publish --provenance --access public
        env:
          NODE_AUTH_TOKEN: ${{ secrets.NPM_TOKEN }}
```

Key points:
- npm provenance (id-token: write) adds verifiable build attestation
- NPM_TOKEN passed as secret (npm doesn't fully support OIDC yet for automation tokens)
- --access public for scoped packages (@fugle/marketdata)
- prepublishOnly runs napi prepublish to generate optionalDependencies structure
