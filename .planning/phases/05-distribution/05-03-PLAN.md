  <files>.github/workflows/build-uniffi.yml</files>
  <action>
Create a reusable workflow (workflow_call) for building UniFFI cdylib.

Platform matrix:
- Linux x64: ubuntu-latest, target x86_64-unknown-linux-gnu, lib: libmarketdata_uniffi.so
- macOS ARM64: macos-latest, target aarch64-apple-darwin, lib: libmarketdata_uniffi.dylib
- macOS x64: macos-latest, target x86_64-apple-darwin, lib: libmarketdata_uniffi.dylib
- Windows x64: windows-latest, target x86_64-pc-windows-msvc, lib: marketdata_uniffi.dll

Steps per matrix job:
1. actions/checkout@v4
2. dtolnay/rust-toolchain@stable with targets: ${{ matrix.target }}
3. Swatinem/rust-cache@v2 with shared-key: uniffi-release
4. cargo build --release -p marketdata-uniffi --target ${{ matrix.target }}
5. actions/upload-artifact@v4:
   - name: uniffi-${{ matrix.target }}
   - path: target/${{ matrix.target }}/release/${{ matrix.lib }}

Output artifact structure for downstream jobs:
- uniffi-x86_64-unknown-linux-gnu/libmarketdata_uniffi.so
- uniffi-aarch64-apple-darwin/libmarketdata_uniffi.dylib
- uniffi-x86_64-apple-darwin/libmarketdata_uniffi.dylib
- uniffi-x86_64-pc-windows-msvc/marketdata_uniffi.dll

Note: macOS requires two separate builds (ARM64 and x64) since we're not creating universal2 for the shared library. C#/Go/Java consumers select the correct library at runtime.
  </action>
