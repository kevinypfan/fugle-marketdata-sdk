  <files>.github/workflows/publish-nuget.yml, bindings/csharp/MarketdataUniffi/MarketdataUniffi.csproj</files>
  <action>
Create a reusable workflow that packs and publishes C# NuGet package with native libraries.

Workflow structure:
```yaml
name: Publish C# to NuGet

on:
  workflow_call:
    secrets:
      NUGET_API_KEY:
        required: true

jobs:
  publish:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4

      - uses: actions/setup-dotnet@v4
        with:
          dotnet-version: '8.0'

      - uses: actions/download-artifact@v4
        with:
          name: uniffi-all
          path: native-libs/

      - name: Copy native libraries to runtimes
        run: |
          mkdir -p bindings/csharp/MarketdataUniffi/runtimes/linux-x64/native
          mkdir -p bindings/csharp/MarketdataUniffi/runtimes/osx-arm64/native
          mkdir -p bindings/csharp/MarketdataUniffi/runtimes/osx-x64/native
          mkdir -p bindings/csharp/MarketdataUniffi/runtimes/win-x64/native
          cp native-libs/linux-x64/*.so bindings/csharp/MarketdataUniffi/runtimes/linux-x64/native/
          cp native-libs/osx-arm64/*.dylib bindings/csharp/MarketdataUniffi/runtimes/osx-arm64/native/
          cp native-libs/osx-x64/*.dylib bindings/csharp/MarketdataUniffi/runtimes/osx-x64/native/
          cp native-libs/win-x64/*.dll bindings/csharp/MarketdataUniffi/runtimes/win-x64/native/

      - name: Pack NuGet
        working-directory: bindings/csharp/MarketdataUniffi
        run: dotnet pack -c Release -o nupkg

      - name: Push to NuGet
        run: dotnet nuget push bindings/csharp/MarketdataUniffi/nupkg/*.nupkg --api-key ${{ secrets.NUGET_API_KEY }} --source https://api.nuget.org/v3/index.json --skip-duplicate
```

Update .csproj to include runtimes from local paths (for CI build):
```xml
<ItemGroup>
  <None Include="runtimes/**/*" Pack="true" PackagePath="runtimes/" />
</ItemGroup>
```

This replaces the conditional Exists() checks with explicit local paths that work in CI.

<task type="auto">
  <name>Task 2: Create Java publish workflow with GitHub Packages</name>
  <files>.github/workflows/publish-java.yml, bindings/java/build.gradle.kts</files>
  <action>
Create a reusable workflow that publishes Java package to GitHub Packages (simpler than Maven Central).

Workflow structure:
```yaml
name: Publish Java to GitHub Packages

on:
  workflow_call:

jobs:
  publish:
    runs-on: ubuntu-latest
    permissions:
      packages: write
      contents: read
    steps:
      - uses: actions/checkout@v4

      - uses: actions/setup-java@v4
        with:
          distribution: 'temurin'
          java-version: '21'

      - uses: actions/download-artifact@v4
        with:
          name: uniffi-all
          path: native-libs/

      - name: Copy native libraries to resources
        run: |
          mkdir -p bindings/java/src/main/resources/native/linux-x64
          mkdir -p bindings/java/src/main/resources/native/osx-arm64
          mkdir -p bindings/java/src/main/resources/native/osx-x64
          mkdir -p bindings/java/src/main/resources/native/win-x64
          cp native-libs/linux-x64/*.so bindings/java/src/main/resources/native/linux-x64/
          cp native-libs/osx-arm64/*.dylib bindings/java/src/main/resources/native/osx-arm64/
          cp native-libs/osx-x64/*.dylib bindings/java/src/main/resources/native/osx-x64/
          cp native-libs/win-x64/*.dll bindings/java/src/main/resources/native/win-x64/

      - name: Publish to GitHub Packages
        working-directory: bindings/java
        run: ./gradlew publish
        env:
          GITHUB_TOKEN: ${{ secrets.GITHUB_TOKEN }}
```

Update build.gradle.kts to add GitHub Packages publishing:
```kotlin
plugins {
    `java-library`
    `maven-publish`
}

publishing {
    publications {
        create<MavenPublication>("gpr") {
            from(components["java"])
            groupId = "tw.com.fugle"
            artifactId = "marketdata-java"
        }
    }
    repositories {
        maven {
            name = "GitHubPackages"
            url = uri("https://maven.pkg.github.com/fugle/fugle-marketdata-sdk")
            credentials {
                username = System.getenv("GITHUB_ACTOR")
                password = System.getenv("GITHUB_TOKEN")
            }
        }
    }
}
```

Note: GitHub Packages requires GITHUB_TOKEN (automatic) instead of Maven Central's GPG signing complexity.
