  <action>
Create a reusable workflow (workflow_call) for building Node.js native addons using napi-rs.

Platform matrix (matching js/package.json napi.targets):
- macOS x64: macos-latest, target x86_64-apple-darwin
- macOS ARM64: macos-latest, target aarch64-apple-darwin
- Linux x64: ubuntu-latest, target x86_64-unknown-linux-gnu
- Linux ARM64: ubuntu-latest, target aarch64-unknown-linux-gnu
- Windows x64: windows-latest, target x86_64-pc-windows-msvc

Workflow inputs:
- version (string): Version for artifact naming

Steps per matrix job:
1. actions/checkout@v4
2. actions/setup-node@v4 with node-version: 20
3. dtolnay/rust-toolchain@stable with targets: ${{ matrix.target }}
4. Swatinem/rust-cache@v2 with shared-key: nodejs-release
5. npm ci (in js/ directory)
6. npm run build -- --target ${{ matrix.target }} (builds with napi-rs)
7. actions/upload-artifact@v4:
   - name: bindings-${{ matrix.target }}
   - path: js/*.node

For Linux ARM64 cross-compilation:
- Use cross-compilation setup from napi-rs template
- Add: sudo apt-get install gcc-aarch64-linux-gnu
- Set: CC=aarch64-linux-gnu-gcc for the build step

Windows and macOS builds run natively (no cross-compilation needed for their respective targets).
  </action>

Review workflow syntax is valid YAML. Check that matrix covers all 5 targets from package.json napi.targets.
  </verify>

build-nodejs.yml exists with 5 matrix entries matching package.json napi.targets, uploads .node artifacts.
  </done>
</task>
