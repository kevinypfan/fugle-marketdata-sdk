  - MarketDataError enum for error handling

### Channel Wrapper (206 lines)
- **MessageChannel**: Buffered channel infrastructure
  - `chan StreamMessage` for message delivery
  - `chan error` for error propagation
  - Thread-safe close with sync.Once
- **StreamingClient**: High-level idiomatic API
  - Connect/Subscribe/Unsubscribe/Close lifecycle
  - Messages() returns `<-chan StreamMessage` for range iteration
  - Errors() for async error monitoring
- **channelListener**: Bridges WebSocketListener callbacks to channels

- **cgo.go**: Linker flags for libmarketdata_uniffi.dylib
  - LDFLAGS for library linking
  - Darwin/Linux rpath configuration

## Key Patterns

### REST API Pattern
```go
// Go idiomatic (T, error) via (*Type, *MarketDataError)
client, err := marketdata_uniffi.NewRestClientWithApiKey(apiKey)
if err != nil {
    log.Fatal(err)
}

quote, err := client.Stock().Intraday().GetQuote("2330")
if err != nil {
    log.Fatal(err)
}
fmt.Println(quote.LastPrice)
```

### WebSocket Channel Pattern
```go
// Idiomatic Go with range loop
client, _ := marketdata_uniffi.NewStreamingClient(apiKey, 100)
defer client.Close()

client.Connect()
client.Subscribe("trades", "2330")

for msg := range client.Messages() {
    fmt.Printf("%s: %s\n", *msg.Symbol, msg.Event)
}
```

### WebSocket Callback Pattern
```go
// Direct callback pattern (also available)
type myListener struct{}

func (l *myListener) OnMessage(msg StreamMessage) {
    fmt.Printf("Got: %s\n", msg.Event)
}
func (l *myListener) OnConnected()    {}
func (l *myListener) OnDisconnected() {}
func (l *myListener) OnError(msg string) {}

client := marketdata_uniffi.NewWebSocketClient(apiKey, &myListener{})
client.Connect()
```

## Technical Decisions

### Library Mode vs UDL Mode
The project uses proc-macro-based UniFFI (`uniffi::setup_scaffolding!()`), not UDL files. uniffi-bindgen-go's `--library` flag extracts type metadata directly from the compiled cdylib, enabling code generation without a separate UDL definition.

### Channel Buffer Size
Default buffer of 100 messages balances:
- Memory efficiency (bounded buffer)
- Burst handling (queue absorbs short spikes)
- Backpressure signaling (full buffer blocks sender)

Users can customize via NewStreamingClient's bufferSize parameter.

### Error Type as Go error
`*MarketDataError` returned from functions can be used as Go's error interface, providing:
- nil check for success
- Type assertion for specific error handling
- String conversion via error interface
