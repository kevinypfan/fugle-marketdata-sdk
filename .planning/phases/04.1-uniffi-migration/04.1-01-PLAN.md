<task type="auto">
  <name>Task 1: Create typed UDL dictionaries for all models</name>
  <files>uniffi/src/marketdata.udl</files>
  <action>
Replace the current marketdata.udl (which returns JSON strings) with typed dictionaries:

1. **Common types:**
   - `PriceLevel { price: f64, size: i64 }`
   - `TotalStats { trade_value: f64?, trade_volume: i64?, transaction: i64? }`
   - `TradeInfo { price: f64?, size: i64?, time: i64?, serial: i64? }`

2. **REST response models:**
   - `Quote` - All fields from core/src/models/quote.rs (date, symbol, open_price, close_price, bids, asks, etc.)
   - `Ticker` - Symbol info with reference, limit up/down prices
   - `Trade` - Individual trade with price, size, time, serial
   - `TradesResponse` - Contains sequence of Trade
   - `IntradayCandle` - OHLCV candle with timestamp
   - `IntradayCandlesResponse` - Contains sequence of candles
   - `VolumeAtPrice` - Price level volume aggregation
   - `VolumesResponse` - Contains sequence of volume levels

3. **WebSocket message models:**
   - `StreamMessage { event: string, channel: string, symbol: string, data: DataPayload? }`
   - `DataPayload` as dictionary with common streaming fields

4. **Error enum (flat, per CONTEXT.md):**
   ```
   [Error]
   enum MarketDataError {
       "NetworkError",
       "AuthError",
       "RateLimitError",
       "InvalidSymbol",
       "ParseError",
       "TimeoutError",
       "WebSocketError",
       "ClientClosed",
       "Other"
   };
   ```

5. **Keep namespace and factory functions** but update return types to use typed models instead of JSON strings.

Use optional types (f64?, i64?, string?) where core models have Option<T>.
Use sequence<T> for Vec<T> fields like bids/asks.
  </action>
  <verify>

    `grep "sequence<" uniffi/src/marketdata.udl` shows collection fields
  </verify>

</task>
