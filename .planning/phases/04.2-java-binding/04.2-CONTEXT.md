# Phase 4.2: Java Binding via UniFFI - Context

**Gathered:** 2026-01-31
**Status:** Ready for planning

<domain>
## Phase Boundary

Generate Java bindings using uniffi-bindgen-java from existing UniFFI proc-macro interface. Provide idiomatic Java API with CompletableFuture async, proper exception handling, and convenient consumption patterns. WebSocket uses callback interface implementing WebSocketListener trait.

</domain>

<decisions>
## Implementation Decisions

### Async Pattern
- Dual sync/async methods: `getQuote()` returns blocking, `getQuoteAsync()` returns `CompletableFuture<T>`
- WebSocket supports both patterns: Listener interface (callback-based) AND BlockingQueue (pull-based)
- Standard Java `CompletableFuture` exception semantics — `ExecutionException` wraps cause, users call `getCause()`
- Timeout configuration at Claude's discretion based on UniFFI capabilities

### Exception Hierarchy
- Two-level hierarchy: `FugleException` base with `ApiException` and `AuthException` subtypes
- Match C# pattern established in Phase 4.1 for consistency across bindings
- Rate limit errors include `getRetryAfterSeconds()` for backoff guidance
- Checked vs unchecked: Claude's discretion (modern Java prefers unchecked/RuntimeException)
- Error message detail: Claude's discretion (balance debugging needs vs user-friendliness)

### Package Structure
- Naming consistent with other bindings (Python: `fugle-marketdata`, Node: `@fugle/marketdata`)
- Gradle with Kotlin DSL as build system
- Minimum Java 11 (LTS baseline)
- Native libraries bundled inside JAR, extracted at runtime (single artifact distribution)

### API Style
- Builder pattern for client construction: `FugleRestClient.builder().apiKey(key).build()`
- Immutable model classes with final fields, no setters (thread-safe, functional style)
- `Optional<T>` for nullable response fields
- JSON library: Claude's discretion based on UniFFI integration needs

### Claude's Discretion
- Timeout configuration approach (per-call vs client-level)
- Checked vs unchecked exception decision
- Error message verbosity and detail level
- JSON library selection (Jackson vs Gson)

</decisions>

<specifics>
## Specific Ideas

- Follow same pattern as C# and Go bindings from Phase 4.1 — uniffi-bindgen-java uses same library mode
- WebSocketListener interface mirrors the foreign trait pattern from uniffi crate
- Match FubonNeo SDK patterns where applicable for familiarity
- Java 11 baseline means no `record` keyword — use immutable classes with Lombok or manual implementation

</specifics>
