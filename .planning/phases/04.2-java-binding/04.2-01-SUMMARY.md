## Decisions Made

**1. Java 21 minimum version requirement**
- **Rationale:** UniFFI-generated code uses pattern matching in switch expressions (Java 21 feature)
- **Impact:** Higher baseline than typical Java libraries, but aligns with modern Java ecosystem
- **Alternative considered:** Post-process generated code to use if-else chains (rejected - brittle)

**2. uniffi.toml for package configuration**
- **Rationale:** uniffi-bindgen-java doesn't support --package flag, requires config file
- **Pattern:** Same configuration approach used for C# (uniffi-bindgen-cs visibility post-processing)

**3. Gradle wrapper for build reproducibility**
- **Rationale:** Ensures consistent Gradle version across environments
- **Version:** 8.5 (latest stable at time of implementation)
