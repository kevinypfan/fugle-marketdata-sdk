## What Was Built

### 1. FugleWebSocketClient with Dual Streaming Patterns

**Callback Mode (Push-Based):**
```java
WebSocketListener listener = new WebSocketListener() {
    public void onConnected() { ... }
    public void onMessage(StreamMessage msg) { ... }
};

try (FugleWebSocketClient client = FugleWebSocketClient.builder()
        .apiKey("...")
        .stock()
        .listener(listener)
        .build()) {
    client.connect().get();
    client.subscribe("trades", "2330").get();
}
```

**Pull Mode (Queue-Based):**
```java
try (FugleWebSocketClient client = FugleWebSocketClient.builder()
        .apiKey("...")
        .stock()
        .queueCapacity(1000)
        .build()) {
    client.connect().get();
    client.subscribe("trades", "2330").get();

    StreamMessage msg = client.poll(1, TimeUnit.SECONDS);
    if (client.hasErrors()) {
        String error = client.pollError();
    }
}
```

**Key Features:**
- Builder pattern: `apiKey()`, `stock()`, `futopt()`, `listener()`, `queueCapacity()`
- Dual modes: Callback (traditional) vs Pull (modern reactive)
- Error queue: `pollError()`, `hasErrors()` for async error monitoring
- AutoCloseable for try-with-resources
- Delegates to generated `WebSocketClient` from UniFFI

## Technical Implementation

### BlockingQueue Pattern

**InternalListener** forwards events to queues:
```java
private static class InternalListener implements WebSocketListener {
    private final BlockingQueue<StreamMessage> messageQueue;
    private final BlockingQueue<String> errorQueue;

    public void onMessage(StreamMessage message) {
        messageQueue.offer(message);  // Non-blocking
    }

    public void onError(String errorMessage) {
        errorQueue.offer(errorMessage);
    }
}
```

**Consumer API:**
- `poll()` - Non-blocking check
- `poll(timeout, unit)` - Wait with timeout
- `take()` - Blocking wait
- `queueSize()` - Current queue depth
- `hasErrors()` / `pollError()` - Error queue access

- Separate tags (`@Tag("integration")`) for CI filtering

**CI Integration:**
- Java 21 required (uniffi-bindgen-java generates Java 21 code)
- Integration tests only on main branch with secrets
