<task type="auto">
  <name>Task 1: Install uniffi-bindgen-java and generate bindings</name>
  <files>
    bindings/java/src/main/java/tw/com/fugle/marketdata/generated/
    Makefile
  </files>
  <action>
1. Install uniffi-bindgen-java from IronCoreLabs:
   ```bash
   cargo install uniffi-bindgen-java --git https://github.com/IronCoreLabs/uniffi-bindgen-java
   ```

2. Ensure the UniFFI cdylib is built:
   ```bash
   cargo build -p marketdata-uniffi --release
   ```

3. Create the bindings directory structure:
   ```bash
   mkdir -p bindings/java/src/main/java/tw/com/fugle/marketdata/generated
   ```

4. Generate Java bindings using library mode (same pattern as C# and Go):
   ```bash
   uniffi-bindgen-java --library target/release/libmarketdata_uniffi.dylib \
     --out-dir bindings/java/src/main/java/tw/com/fugle/marketdata/generated \
     --package tw.com.fugle.marketdata.generated
   ```

5. Add gen-java target to Makefile following the C#/Go pattern:
   - Build the cdylib first
   - Generate bindings with library mode
   - Include help entry

The --package flag sets the Java package name. If uniffi-bindgen-java uses a different flag syntax, adapt accordingly (check --help).

Note: The generated code will use JNA for native access and CompletableFuture for async methods. This is different from the official Kotlin bindings which use coroutines.
  </action>
  <verify>
    - `which uniffi-bindgen-java` shows installed binary
    - `ls bindings/java/src/main/java/tw/com/fugle/marketdata/generated/*.java` shows generated files
    - `make gen-java` runs without errors
  </verify>
  <done>
    uniffi-bindgen-java installed, Java bindings generated with all model types (Quote, Ticker, etc.), REST client classes, WebSocket client, and MarketDataException
  </done>
</task>

1. Create settings.gradle.kts:


2. Create gradle.properties:
   ```properties
   org.gradle.jvmargs=-Xmx1024m
   ```

3. Create build.gradle.kts following RESEARCH.md patterns:
   ```kotlin
   plugins {
       `java-library`
   }

   group = "tw.com.fugle"
   version = "0.2.0"

   java {
       toolchain {
           languageVersion.set(JavaLanguageVersion.of(11))
       }
   }

   repositories {
       mavenCentral()
   }

   dependencies {
       // JNA required for UniFFI-generated bindings
       implementation("net.java.dev.jna:jna:5.14.0")

       // Testing
       testImplementation("org.junit.jupiter:junit-jupiter:5.10.0")
   }

   tasks.test {
       useJUnitPlatform()
   }

   // Source sets - generated code lives in generated/ package
   sourceSets {
       main {
           java {
               srcDirs("src/main/java")
           }
       }
   }
   ```

4. Initialize Gradle wrapper (if not present):
   ```bash
   cd bindings/java && gradle wrapper --gradle-version 8.5
   ```

5. Test compilation:

   ```bash
   cd bindings/java && ./gradlew compileJava
   ```

If compilation fails due to visibility issues (like C# had), note what post-processing is needed for Plan 02.

   ```makefile
   build-java: gen-java
   ```
