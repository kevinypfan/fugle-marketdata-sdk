# Coding Conventions

**Analysis Date:** 2026-01-30

## Naming Patterns

**Files:**
- Rust: snake_case with `.rs` extension - `runtime.rs`, `rest.rs`, `websocket.rs`
- Python: snake_case with `.py` extension - `test_rest.py`, `test_websocket.py`
- JavaScript: camelCase or snake_case depending on module - `index.js`, `test_rest.js`
- Module organization: Group related functionality (e.g., `websocket/connection.rs`, `rest/auth.rs`)

**Functions:**
- Rust: snake_case for all functions - `create_runtime()`, `block_on()`, `test_runtime_creation()`
- Python: snake_case for functions and methods - `test_import()`, `test_client_creation()`, `run_all_tests()`
- JavaScript: camelCase for methods - `isConnected()`, `subscribe()`, accessed as `client.stock.intraday.quote()`

**Variables:**
- Rust: snake_case for locals and mutable bindings - `let runtime`, `let counter`, `mut handles`
- Python: snake_case for all variables - `messages_received = []`, `client1 = RestClient()`
- JavaScript: camelCase for locals - `const client`, `const stockIntraday`, `const futopt`

**Types:**
- Rust: PascalCase for structs, traits, and enums - `AsyncRuntime`, `MarketDataError`, `Quote`, `RestClient`
- Rust: Derive attributes commonly used: `#[derive(Debug, Clone, Default, Serialize, Deserialize)]`
- Python: Classes in PascalCase - `RestClient`, `WebSocketClient`
- JavaScript/TypeScript: PascalCase for classes - `RestClient`, `StockClient`, `StockIntradayClient`

## Code Style

**Formatting:**
- Rust: 4-space indentation (standard Rust style via rustfmt)
- Python: 4-space indentation (PEP 8 standard)
- JavaScript: 4-space indentation (inferred from test files)
- Line wrapping: Long lines broken logically with clear indentation

**Linting:**
- Rust: rustfmt for formatting, clippy for linting (enforced at build time)
- Python: No explicit linter configured; code follows PEP 8 conventions
- JavaScript: ESLint disabled in generated files (see `/* eslint-disable */` in `index.js`), auto-generated code by NAPI-RS
- Note: `js/index.js` is auto-generated by NAPI-RS with linting directives disabled

**Comments and Documentation:**
- Rust: Doc comments using `///` for public items with markdown formatting
  ```rust
  /// Create a new REST client with API key authentication
  ///
  /// @param apiKey - Your Fugle API key
  pub fn new(api_key: String) -> Self { }
  ```
- Rust: Module-level doc comments `//!` at file start explaining module purpose
  ```rust
  //! REST client module for Fugle marketdata API
  //! This module provides authentication mechanisms and HTTP client
  ```
- Python: Docstrings with triple quotes for functions and modules
  ```python
  """
  Test script for marketdata_py REST client

  This script tests:
  1. Module import
  2. RestClient creation
  """
  ```
- JavaScript: JSDoc-style comments for methods and classes
  ```javascript
  /**
   * Get intraday quote for a stock symbol
   *
   * @param symbol - Stock symbol (e.g., "2330" for TSMC)
   * @returns Quote object with current price
   */
  ```

## Import Organization

**Rust Order:**
1. Standard library imports (`use std::...`)
2. External crate imports (`use tokio::...`, `use serde::...`)
3. Internal crate imports (`use crate::...`)
4. Re-exports via `pub use`

Example from `lib.rs`:
```rust
pub mod errors;
pub mod models;
pub mod rest;
pub mod websocket;

pub use errors::MarketDataError;
pub use rest::{Auth, RestClient};
pub use websocket::{ConnectionConfig, ...};
```

**Python Order:**
1. Standard library (`import sys`, `import gc`)
2. External packages (none in test files)
3. Local imports (none in test files - uses installed package)

Example from `test_rest.py`:
```python
import marketdata_py
from marketdata_py import RestClient, MarketDataError
```

**JavaScript Order:**
1. Core modules (`const fs = require('fs')`, `const path = require('path')`)
2. Local imports (`const { RestClient } = require('./index.js')`)

Example from `test_rest.js`:
```javascript
const { RestClient } = require('./index.js');
```

## Error Handling

**Rust Pattern:**
- Use `thiserror::Error` derive macro for enum-based error types
- Custom error types with structured fields and formatted messages
- File: `core/src/errors.rs`

Example:
```rust
#[derive(Error, Debug)]
pub enum MarketDataError {
    #[error("Invalid symbol: {symbol}")]
    InvalidSymbol { symbol: String },

    #[error("Authentication error: {msg}")]
    AuthError { msg: String },
}

// Implement From for conversion
impl From<serde_json::Error> for MarketDataError {
    fn from(err: serde_json::Error) -> Self {
        MarketDataError::DeserializationError { source: err }
    }
}
```

- Error code ranges (documented in `errors.rs`):
  - 1000-1999: Client errors (bad input, deserialization)
  - 2000-2999: Server/API errors (auth, connection, HTTP)
  - 3000-3999: Network errors (timeout, WebSocket)
  - 9000-9999: Internal errors (unexpected failures)

**Python Pattern:**
- Custom exception with message and error code in args
- File: `test_rest.py` demonstrates usage

Example:
```python
try:
    raise MarketDataError("Test error", 1001)
except MarketDataError as e:
    assert str(e.args[0]) == "Test error"
    assert e.args[1] == 1001  # error_code in args[1]
```

**JavaScript Pattern:**
- Error objects with message property
- File: `test_rest.js` shows error handling

Example:
```javascript
try {
    const result = client.stock.intraday.quote('2330');
} catch (err) {
    const hasErrorCode = err.message.match(/\[\d+\]/);
    console.log('Error message:', err.message);
}
```

## Logging

**Framework:** No explicit logging framework in SDK code; test files use `console.log()` (JavaScript) and `print()` (Python).

**Patterns:**
- Rust: FFI boundary errors logged to stderr via `eprintln!()` in `runtime.rs`
  ```rust
  eprintln!("PANIC: Caught panic at FFI boundary");
  eprintln!("Failed to create runtime: {}", e);
  ```
- Python: Print statements for test output and status
  ```python
  print("Testing import...")
  print(f"  API key client: {client1}")
  print("  OK\n")
  ```
- JavaScript: Console logging for test output
  ```javascript
  console.log('=== marketdata-js REST Client Tests ===\n');
  console.log('  - RestClient class:', typeof RestClient === 'function' ? 'OK' : 'FAIL');
  ```

## Module Design

**Exports - Rust:**
- Use `pub use` in parent module to re-export commonly-used types
- File: `core/src/lib.rs` shows main re-export pattern

Example:
```rust
// Re-export error types
pub use errors::MarketDataError;

// Re-export REST client types
pub use rest::{Auth, RestClient};

// Re-export WebSocket types
pub use websocket::{ConnectionConfig, ...};
```

**Exports - Python:**
- Package-level exports handled at binding layer (generated by PyO3)
- File: `test_rest.py` imports like `from marketdata_py import RestClient, MarketDataError`

**Exports - JavaScript:**
- CommonJS module.exports pattern (auto-generated by NAPI-RS)
- File: `index.js` exports platform-specific native bindings

Example:
```javascript
const { RestClient, StockClient, WebSocketClient, ... } = nativeBinding
module.exports.RestClient = RestClient
module.exports.StockClient = StockClient
```

**Barrel Files:** Not used in this codebase; re-exports done via `pub use` statements in module `mod.rs` files.

## Attribute Macros

**Rust Derive Attributes:**
- Standard derives: `#[derive(Debug, Clone, Default, Serialize, Deserialize)]`
- Feature-gated derives for Python/JavaScript bindings:
  ```rust
  #[cfg_attr(feature = "python", pyo3::prelude::pyclass)]
  #[cfg_attr(feature = "js", napi_derive::napi(object))]
  pub struct Quote { }
  ```
- Serde field attributes for API mapping:
  ```rust
  #[serde(rename = "lastPrice")]
  pub last_price: Option<f64>,

  #[serde(default)]
  pub bids: Vec<PriceLevel>,
  ```

**Function Attributes:**
- FFI-safe functions marked with `#[no_mangle]` and `extern "C"` signature
  ```rust
  #[no_mangle]
  pub extern "C" fn create_runtime() -> *mut AsyncRuntime { }
  ```

## Special Patterns

**FFI Safety (Rust):**
- Panic catching macros defined in `runtime.rs` for FFI boundary safety
- Converts panics to null pointers or void returns
- All FFI functions wrapped with `ffi_catch_ptr!` or `ffi_catch_void!`

**Async/Await (Rust):**
- Single runtime instance pattern using `tokio::runtime::Runtime`
- Block-on wrapper for synchronous calling from FFI
- Spawn pattern for fire-and-forget tasks

**Client Chain Pattern (All Languages):**
- Fluent interface for accessing nested clients
- Rust: Property getters (`pub fn stock(&self) -> StockClient`)
- Python: Property access (`client.stock`)
- JavaScript: Getter properties (`get stock()`)

Example:
```rust
client.stock.intraday.quote("2330")
client.futopt.intraday.products("FUTURE")
```
